┌─────────────────────────────────────────────────────────────┐
│                    EDIT MODE FLOW                           │
└─────────────────────────────────────────────────────────────┘

1. VIEW MODE (Default)
   └─ [Edit Profile] button visible

2. CLICK "Edit Profile"
   └─ All fields become input boxes
   └─ Header shows [Save Changes] [Cancel] buttons
   └─ "Back to Dashboard" button hidden

3. MODIFY FIELDS
   └─ Type in any input field
   └─ Changes stored in temporary state

4. CLICK "Save Changes"
   └─ Show loading spinner
   └─ Send PUT request to /api/user/profile
   └─ Display success/error toast
   └─ Update local state with saved data
   └─ Return to view mode

5. OR CLICK "Cancel"
   └─ Discard all changes
   └─ Revert to original data
   └─ Return to view mode


┌─────────────────────────────────────────────────────────────┐
│                    TECHNICAL DETAILS                        │
└─────────────────────────────────────────────────────────────┘

Frontend State Management:
- isEditing: boolean (edit mode toggle)
- doctor: original data from backend
- editedDoctor: temporary edited data
- saving: boolean (loading state)

API Calls:
- GET /api/user (fetch profile on load)
- PUT /api/user/profile (save changes)

Form Fields:
- Text inputs: name, email, phone, specialization, department,
  qualification, experience, availableDays, availableTimings
- Textarea: about/professional summary

Validation:
- Email uniqueness check on backend
- Required field validation
- Error handling for network issues
